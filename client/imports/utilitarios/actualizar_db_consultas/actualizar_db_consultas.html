Esta función permite al usuario actualizar la base de datos de consultas, con los registros que se han agregado/editado/eliminado más recientemente.

La base de datos de consultas permite al usuario de este programa, scrwebm, hacer consultas a los datos que se han registrado, como: riesgos, contratos, remesas, etc., desde un programa separado y diferente.

En el programa de consultas se implementan nuevas formas de hacer consultas a los datos, de una forma (que intenta ser) más amigable, simple y efectiva.

Para que las consultas, desde este programa separado de consultas, puedan ser llevadas a cabo, los registros que hace el usuario deben ser copiados y actualizados en forma permanente. Este es, justamente, el objetivo de esta función: mantener la base de datos de consultas adecuadamente actualizada.

Esta función permite al usuario reiniciar el proceso de copia de registros a la base de datos de consultas.

Al ejecutar este proceso, la fecha de copia más reciente es eliminada. Al hacerlo, la próxima vez que la copia sea efectuada, será como si fuese la primera vez que se esté haciendo. El resultado será que todos los registros volverán a ser copiados (refrescados).

Este proceso puede ser útil cuando el usuario desee asegurarse que todos los registros, en la base de datos de consultas, sean refrescados (actualizados); es decir, que la copia de estos se efectúe nuevamente para asegurarnos que sus valores sean actualizados.

Como resultado de ejecutar esta función, la próxima copia que se efectúe será siempre bastante lenta, pues todos los registros serán refrescados en la base de datos de consultas, y no resultarán copiados solo los que se han editado en forma muy reciente.
