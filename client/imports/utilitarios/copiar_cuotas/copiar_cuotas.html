Esta función le permite al usuario copiar las cuotas, desde la base de datos principal del programa, a la base de datos de consultas.

La base de datos de consultas permite al usuario de este programa, scrwebm, hacer consultas a los datos que se han registrado, como: riesgos, contratos, remesas, etc., desde un programa separado y diferente.

En el programa de consultas se implementan nuevas formas de hacer consultas a los datos, de una forma (que intenta ser) más amigable, simple y efectiva.

Para que las consultas, desde este programa separado de consultas, puedan ser llevadas a cabo, los diferentes registros que hace el usuario, como: riesgos, contratos, cuotas, etc., deben ser copiados y actualizados en forma permanente. Este es, justamente, el objetivo de esta función: mantener la base de datos de consultas adecuadamente actualizada, con las cuotas de cobro/pago que el usuario va registrando y actualizando a medida que registra las diversas transacciones que conforman el manejo de este programa.

Esta función permite al usuario reiniciar el proceso de copia de cuotas de cobro/pago a la base de datos de consultas.

El resultado de ejecutar este proceso es que la próxima vez que se efectúe la copia, todas las cuotas registradas serán copiadas nuevamente.

Este proceso puede ser útil cuando el usuario desee asegurarse que todas las cuotas, en la base de datos de consultas, sean refrescadas (actualizadas); es decir, que la copia de estas cuotas se efectúe nuevamente para asegurarnos que sus valores sean actualizados (y correctos).
