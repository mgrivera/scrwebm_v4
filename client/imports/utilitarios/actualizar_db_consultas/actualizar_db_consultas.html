Esta función permite al usuario copiar los catálogos, como asegurados, ramos, monedas, etc., a la base de datos de consultas. Esta función copiará solo los registros que se han agregado/editado/eliminado recientemente. Un registro que fue copiado antes no volverá a ser copiado, a menos que haya recibido modificaciones por parte del usuario.

La base de datos de consultas permite al usuario de este programa, scrwebm, hacer consultas a los datos que se han registrado, como: riesgos, contratos, remesas, etc., desde un programa separado y diferente.

En el programa de consultas se implementan nuevas formas de hacer consultas a los datos, de una forma (que intenta ser) más amigable, simple y efectiva.

Para que las consultas, desde este programa separado de consultas, puedan ser llevadas a cabo, los registros que hace el usuario deben ser copiados y actualizados en forma permanente.

Esta función permite al usuario reiniciar el proceso de copia de catálogos a la base de datos de consultas.

Al ejecutar este proceso, todos los registros que copia este proceso, serán copiados nuevamente, cuando el usuario ejecute próximamente el proceso.

Este proceso puede ser útil cuando el usuario desee asegurarse que todos los catálogos, en la base de datos de consultas, sean refrescados (actualizados); es decir, que la copia de estos se efectúe nuevamente para asegurarnos que sus valores sean actualizados.

Como resultado de ejecutar esta función, la próxima copia que se efectúe será siempre bastante lenta, pues todas las tablas de tipo Catálogo serán vueltas a copiar.
